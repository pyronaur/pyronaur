# AGENTS

## GitHub README Playbook

- Keep `README.md` short and public-facing.
- Do not reference external personal profile templates.
- For the projects list, include only non-fork repos with recent `pyronaur` commits.
- For README refreshes, derive recency cutoff from the last `README.md` update commit timestamp (for example `git log -1 --format=%cI -- README.md`) and include only repos with qualifying `pyronaur` commits newer than that baseline.
- Keep internal filtering logic and cutoff details out of public README text.
- After edits, verify links and list contents.
